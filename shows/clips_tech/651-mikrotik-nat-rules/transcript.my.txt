port 80 နှင့် port 443 တို့ကို port 80 ခုဖွင့်ထားခြင်းသည်လုံခြုံမှုမရှိသော HTTP ဖြစ်သည်
port and port 443 သည် SSL encrypted HTTP port ဖြစ်ပြီး၎င်းနှစ်ခုလုံးကိုကျွန်ုပ်လိုချင်သည်
တကယ်လို့တစ်ယောက်ယောက်ကတကယ်တော့ HTTP colon ကိုမျဉ်းစောင်းမရိုက်ဘဲနေတယ်ဆိုရင်
slash studio dot အမျိုးအစား ၅ ခုကို slash စတူဒီယိုက port 80 နှင့်ကိုနှိပ်ပါ
ငါ port 80 ဖွင့်လှစ်ရှိသည်မဟုတ်ခဲ့ပါလျှင် 443 သို့အလိုအလျောက် redirect
သူတို့တစ်တွေဆာဗာကိုရရုံအမှားဒါ backing ကိုရလိမ့်မယ်လို့ကြောင်း redirect ရဘယ်တော့မှ
ကျွန်တော် web fig ကိုသုံးနေသည့်ကျွန်ုပ်တို့၏ mikrotik configuration ကိုသွားရန်လိုအပ်သည်
ထိပ်ဆုံးရှိ NAT tab ကိုနှိပ်ပြီးအသစ်တစ်ခုကိုဖန်တီးပါ။ firewall ပေါ်တွင် IP ကိုနှိပ်ပါ
NAT rule က NAT rule သည်၎င်းကိုသင်မည်သည့်နေရာတွင်သွားချင်သည်ကိုပြောပြလိမ့်မည်
ဒီ port နဲ့သင်ဘာလုပ်ချင်လဲ
ဒါပေမယ့်တကယ်တော့သူကဖွင့်ဖွင့်တော့မှာမဟုတ်ပါဘူး
ယခု firewall rule ကိုရောက်လာပြီ၊ ဒါကြောင့်ထို IP ၏ firewall rules ကိုနှိပ်ပါ
Firewall နဲ့လက်ခံဖို့သွားတဲ့ firewall rules အသစ်ကိုလုပ်ဖို့လိုတယ်
အဲဒီဆက်သွယ်မှုကိုဖြတ်ပြီးသူတို့အားဆက်သွယ်မှုများကိုခွင့်ပြုပါနှင့်သင်နောက်ထပ်နိုင်ပါတယ်
အိုင်ပီလိပ်စာများ (သို့) အိုင်ပီရင်းမြစ်အုပ်စုများကို ၀ င်ရောက်ရှာဖွေပါ
ယနေ့ကျွန်ုပ်တို့မဖုံးလွှမ်းနိုင်သော်လည်းသင်ရရှိနိုင်ပါသည်
ဒါကကျွန်တော်တို့ကိုအများကြီးပြင်ဆင်မှုနှင့်ထိန်းချုပ်မှုတွေအများကြီးပေးလိမ့်မယ်ဆိုတဲ့စိတ်ကူးပါ
ကျွန်ုပ်တို့၏ကွန်ယက်များမှတဆင့်မည်ကဲ့သို့အသွားအလာလမ်းကြောင်းကိုဖြတ်သန်းသွားသည်သာမကမည်သည့်အရာနှင့် IP လဲပါ ၀ င်သည်
လိပ်စာများနှင့်ကျွန်ုပ်တို့၏ကွန်ယက်မှတဆင့်မည်သည့်ကွန်ယက်များကိုချိတ်ဆက်နိုင်မည်နည်း
