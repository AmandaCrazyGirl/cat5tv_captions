ရုပ်ပိုင်းဆိုင်ရာအားဖြင့် HTTP အူမကြီး၏အူမကြီးစလတ်စ်ရှယ်ရှယ်အမျိုးအစားအမျိုးအစား ၅ တွင်တီဗွီရိုက်ပါ
အဲဒီအစားငါ port 80 ကိုနှိပ်ပါနှင့် 443 အလိုအလျှောက် redirect လိမ့်မယ်
port 80 မဖွင့်ထားဘူးဆိုတော့သူတို့က redirect မရတော့ဘူး
ဆာဗာတစ်ခုမှအမှားမတွေ့ပါ၊ ထို့ကြောင့်အရန်ကူးယူခြင်းအားဖြင့်ကျွန်ုပ်တို့၏ mikrotik သို့သွားရန်လိုအပ်ပါသည်
configuration ကိုကျွန်တော် web fig ကိုသုံးပြီး firewall ကိုနှိပ်ပါ IP ကိုနှိပ်ပါ
ထိပ်ဆုံးက NAT tab မှာပေါ်လာပြီး NAT rule တစ်ခုပြောပြမယ့် NAT rule အသစ်တစ်ခုကိုဖန်တီးပါ
သင်မည်သည့်နေရာတွင်သင်လုပ်လိုသည်ကိုဤဆိပ်ကမ်းဖြင့်သင်မည်သည့်နေရာတွင်သွားလိုသနည်း
အဲဒါကိုဖွင့်ပေမယ့်တကယ်တော့ဒါဟာအများပြည်သူဖွင့်ဖွင့်လှစ်မယ့်မဟုတ်ပါဘူး
ယခု firewall rule လာသောအခါထို IP ၏ firewall rules ကိုနှိပ်ပါ
Firewall နဲ့လက်ခံဖို့သွားတဲ့ firewall rules အသစ်ကိုလုပ်ဖို့လိုတယ်
အဲဒီဆက်သွယ်မှုကိုဖြတ်ပြီးသူတို့အားဆက်သွယ်မှုများကိုခွင့်ပြုပါနှင့်သင်နောက်ထပ်နိုင်ပါတယ်
အိုင်ပီလိပ်စာများ (သို့) အိုင်ပီရင်းမြစ်အုပ်စုများကို ၀ င်ရောက်ရှာဖွေပါ
ယနေ့ကျွန်ုပ်တို့မဖုံးလွှမ်းနိုင်သော်လည်းသင်ရရှိနိုင်ပါသည်
ဒါကကျွန်တော်တို့ကိုအများကြီးပြင်ဆင်မှုနှင့်ထိန်းချုပ်မှုတွေအများကြီးပေးလိမ့်မယ်ဆိုတဲ့စိတ်ကူးပါ
ကျွန်ုပ်တို့၏ကွန်ယက်များမှမည်ကဲ့သို့အသွားအလာလမ်းကြောင်းကိုဖြတ်သန်းသွားသည်သာမကမည်သူနှင့်မည်သူမည်သည်တို့နှင့်ပါ
မည်သည့် IP လိပ်စာများနှင့်ကျွန်ုပ်တို့၏ကွန်ယက်မှတဆင့်မည်သည့်ကွန်ယက်များချိတ်ဆက်နိုင်မည်၊
